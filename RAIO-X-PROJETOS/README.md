## Critério usado

Cada projeto foi avaliado por:

- Potencial comercial
- Facilidade de começar com baixo custo
- Possibilidade de publicar no GitHub
- Capacidade de virar portfólio
- Possibilidade de gerar venda, lead ou autoridade
- Complexidade técnica
- Próxima ação prática

## Estratégia geral

Não misturar tudo em um único arquivo solto. Cada projeto deve ter:

- README próprio
- pasta separada
- checklist
- roadmap
- arquivos de apoio
- versão mínima publicável

## Regra de ouro

Primeiro publicar versões simples. Depois melhorar.

Site simples online vale mais que projeto perfeito parado.
