# Publicação — Site Danilo A. de Lima

## Como testar localmente

1. Abra a pasta do projeto no VS Code.
2. Instale a extensão **Live Server**.
3. Clique com o botão direito em `index.html`.
4. Escolha `Open with Live Server`.
5. Teste o site no navegador.

## Checklist antes de publicar

- [ ] O site abre sem erro?
- [ ] O menu funciona?
- [ ] Os botões apontam para links reais?
- [ ] As capas dos livros carregam?
- [ ] O texto do autor está revisado?
- [ ] O site funciona no celular?
- [ ] Os links de compra foram conferidos?

## Publicação no GitHub Pages

1. Entre no repositório no GitHub.
2. Clique em **Settings**.
3. Vá em **Pages**.
4. Em **Source**, escolha a branch `main`.
5. Escolha a pasta raiz `/root`.
6. Salve.
7. Aguarde o GitHub gerar o link público.

## Boas práticas

- Testar no celular antes de divulgar.
- Não deixar links `href="#"` em botões principais.
- Não subir imagens muito pesadas.
- Manter README e roadmap atualizados.
