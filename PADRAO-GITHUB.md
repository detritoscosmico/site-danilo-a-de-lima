Usar letras minúsculas, hífens e nomes claros.

## Nomes fora de padrão encontrados

- `-meu-site-`
- `terminal-lobao`

## Ação recomendada

- `-meu-site-` → renomear para `site-laboratorio-danilo` ou arquivar
- `terminal-lobao` → renomear para `sala-de-guerra-terminal-danilo`

## Estrutura mínima de cada repositório

```text
projeto/
├── README.md
├── docs/
│   └── ROADMAP.md
├── src/
├── assets/
└── tests/
```

## README mínimo

Todo repositório precisa explicar:

1. O que é o projeto
2. Para que serve
3. Como rodar
4. Estrutura de pastas
5. Próximos passos
6. Status atual

## Política de organização

- Um projeto grande pode ter repositório próprio.
- Ideias em teste ficam em pasta dentro do hub.
- Produto comercial precisa de página de venda.
- Sistema técnico precisa de documentação.
- Código perigoso ou destrutivo deve ser evitado.
